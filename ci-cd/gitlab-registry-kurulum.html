GitLab Registry Kurulumu ve Yapılandırma Rehberi

GitLab Registry Nedir?

GitLab Registry, GitLab’a entegre bir Docker container registry hizmetidir. Bu özellik, projeleriniz için Docker imajlarını depolamanıza, sürüm yönetimi yapmanıza ve CI/CD süreçlerinde doğrudan kullanmanıza olanak tanır.

GitLab Registry’yi Etkinleştirme

1. GitLab Yapılandırma Dosyasını Düzenleme

GitLab Registry’yi kurmadan önce, GitLab’ın Docker ile çalıştığından emin olun. Aşağıdaki adımlarla GitLab Registry’yi etkinleştirebilirsiniz:

1.1. Yapılandırma Dosyasını Açın

sudo nano /etc/gitlab/gitlab.rb

1.2. Registry Yapılandırmasını Etkinleştirin

Registry'yi etkinleştirmek için aşağıdaki ayarları ekleyin:


registry['enable'] = true
registry_external_url 'https://registry.harunoner.com'
registry_nginx['ssl_certificate'] = "/etc/gitlab/ssl/registry.harunoner.com.crt"
registry_nginx['ssl_certificate_key'] = "/etc/gitlab/ssl/registry.harunoner.com.key"

Burada, registry_external_url kısmı GitLab Registry’nin dışarıdan erişileceği URL’yi belirtir. SSL sertifikaları ile güvenli bağlantılar sağlamak için TLS sertifikası ayarlarını yapmanız gerekebilir.

1.3. Harici Bağlantı Noktasını Yapılandırma

Registry için bağlantı noktasını yapılandırmak isterseniz, şu ayarı ekleyebilirsiniz:


registry['registry_http_addr'] = "0.0.0.0:5005"

2. Yapılandırma Değişikliklerini Uygulama

Yapılandırma dosyasını kaydettikten sonra, GitLab’ı yeniden yapılandırarak değişiklikleri uygulayın:

sudo gitlab-ctl reconfigure

Docker CLI ile GitLab Registry’ye Bağlanma

1. Docker CLI Üzerinden Giriş Yapma

GitLab Registry’yi kullanmaya başlamak için, Docker CLI ile giriş yapmanız gerekir:

docker login registry.harunoner.com

GitLab kullanıcı adı ve kişisel erişim belirteciniz (token) ile giriş yapın.

2. Docker İmajlarını GitLab Registry’ye Gönderme

Docker imajlarını GitLab Registry’ye göndermek için imajı doğru etikete göre yeniden etiketleyin:


docker tag  registry.harunoner.com/group/project:tag
docker push registry.harunoner.com/group/project:tag

GitLab CI/CD ile Otomatik Registry Kullanımı

1. .gitlab-ci.yml Dosyasını Yapılandırma

GitLab CI/CD ile imajları otomatik olarak Registry’ye gönderebilirsiniz. Aşağıdaki gibi bir yapı ekleyerek CI/CD pipeline’ınızı oluşturun:


stages:
  - build
  - deploy

variables:
  IMAGE_TAG: "$CI_REGISTRY_IMAGE:$CI_COMMIT_REF_NAME"

build:
  stage: build
  script:
    - docker build -t $IMAGE_TAG .
    - docker push $IMAGE_TAG

GitLab Registry’nin Avantajları

  • CI/CD entegrasyonu: Docker imajlarınızı doğrudan GitLab pipelineları ile yönetebilirsiniz.
  • Özelleştirilebilir erişim kontrolü: Güvenli bir ortam sağlar.
  • Versiyon kontrolü: Docker imajlarının sürüm takibi kolayca yapılabilir.

Sonuç

GitLab Registry, Docker imajlarınızı GitLab ile kolayca yönetmenize olanak tanır. CI/CD süreçlerinizi optimize ederek projelerinizi daha verimli bir şekilde geliştirebilirsiniz.
