GitLab Container ve Yerel Kurulum Rehberi

GitLab Container Kurulumu (Docker ile)

1. Adım: GitLab Görselini İndirin

GitLab’ın Enterprise veya Community sürümünü Docker ile indirin:

docker pull gitlab/gitlab-ce:latest
docker pull gitlab/gitlab-ee:latest

2. Adım: Container Çalıştırın

Container çalıştırarak veri ve log dosyalarını mount edin:

docker run --detach \
--hostname gitlab.harunoner.com \
--publish 443:443 --publish 80:80 --publish 22:22 \
--name gitlab \
--restart always \
--volume /srv/gitlab/config:/etc/gitlab \
--volume /srv/gitlab/logs:/var/log/gitlab \
--volume /srv/gitlab/data:/var/opt/gitlab \
gitlab/gitlab-ce:latest

3. Adım: GitLab Yapılandırmasını Özelleştirin

GitLab’ın yapılandırma dosyasını /srv/gitlab/config/gitlab.rb üzerinde düzenleyin.

4. Adım: Yapılandırmayı Uygula

docker exec -it gitlab gitlab-ctl reconfigure

GitLab Yerel Kurulumu (Native Installation)

1. Adım: GitLab Depolarını Ekleyin

Debian tabanlı sistemler için GitLab deposunu ekleyin:

curl -s https://packages.gitlab.com/install/repositories/gitlab/gitlab-ee/script.deb.sh | sudo bash

2. Adım: GitLab’ı Kurun

GitLab EE sürümünü kurarak EXTERNAL_URL belirtin:

sudo EXTERNAL_URL="http://gitlab.harunoner.com" apt-get install gitlab-ee

Rocky Linux, Red Hat gibi dağıtımlara kurmak için dnf veya yum paketlerini kullanın:

yum install gitlab-ee
dnf install gitlab-ee

3. Adım: GitLab Yapılandırmasını Güncelleyin

/etc/gitlab/gitlab.rb üzerinde gerekli ayarları yapabilirsiniz.

4. Adım: Yapılandırmayı Uygula

sudo gitlab-ctl reconfigure

Hangi Yöntemi Tercih Etmelisiniz?

Container Kurulumu: İzolasyon sağlar, taşınabilirlik sunar, güncelleme ve yedekleme kolaydır.

Yerel Kurulum: Daha yüksek performans sunar ve tam kontrol sağlar.

Sonuç

Bu rehber, GitLab kurulumları için ihtiyaçlarınıza uygun seçenekleri değerlendirebilmeniz için hazırlanmıştır.
